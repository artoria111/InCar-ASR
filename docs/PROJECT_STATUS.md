# Project status

Updated: 2026-07-23

This document separates code availability from experimentally verified results.

## Reproducibility levels

| Level | Meaning |
| --- | --- |
| Reproducible | Required code and public resources are available with a documented command |
| Partial | Core code exists, but private data, generated artifacts, or hardware are required |
| Planned | Interface, report, or test plan exists without a completed experiment |

## Module status

### 01 — Dataset building: reproducible

Implemented:

- labelled microphone collection;
- conversion to mono PCM WAV;
- segmentation and deterministic splits;
- metadata, duplicate, clipping, weak-signal, and split-leakage checks;
- configurable vehicle-noise categories;
- SNR mixing, speed perturbation, SpecAugment, and simulated reverberation;
- synthetic-audio unit tests.

Not included:

- raw vehicle recordings;
- processed datasets;
- a published dataset checksum manifest.

### 02 — ASR training: partial

Implemented:

- Paraformer-tiny fine-tuning loop;
- frozen-encoder training option;
- CTC and decoder losses;
- relative-path manifest generation and `INCAR_ASR_DATA_ROOT` resolution;
- NumPy feature-cache generation using the shared 80-bin, LFR 7/6 frontend;
- corpus-weighted CER, latency, RTF, error, and bad-case reports;
- one historical model checkpoint.

Blocking reproducibility:

- source audio is not distributed;
- checked-in manifests now contain relative paths and require the matching
  AISHELL or private command-audio root at runtime;
- no verified training cache/checkpoint pair is published;
- the historical CER report cannot be regenerated without the original audio and model.

### CPU inference baseline: reproducible

The repository-level demo uses:

- model: `sherpa-onnx-paraformer-zh-small-2024-03-09`;
- precision: INT8;
- runtime: `sherpa-onnx==1.13.4`;
- source: the official k2-fsa release;
- archive SHA-256:
  `da92b3db5218c5be53aad53e57d1b6e63e7fc98a0e054fbdd6dbe18e9c6b1450`.

Run `make demo`.

This baseline is for repository reproducibility. It does not claim to be the final car-command model or the final Atlas model.

### 03 — Compression and distillation: partial

Implemented:

- KL-divergence logit loss;
- hidden-state MSE loss;
- attention loss;
- official FunASR ONNX export wrapper and export manifest;
- reference/candidate ONNX numeric parity report;
- an evidence-aware distillation report generator that refuses placeholders
  in verification mode.

Not yet verified:

- a full teacher-to-student training run;
- AMCT INT8 quantization;
- OM conversion and post-quantization CER.

### 04 — Atlas deployment: partial

Implemented:

- CMake project and shared library;
- AscendCL lifecycle and OM execution wrapper;
- WAV CLI;
- VAD, FBank, and decoder components;
- FFT-based FBank with LFR 7/6;
- adaptive-energy VAD;
- direct Paraformer and optional CTC repeat decoding;
- reusable AscendCL buffers with explicit lifecycle cleanup;
- fixed-shape ATC conversion, checksummed artifact manifests, profiling, and
  board-report scripts;
- ACL link smoke test and hardware-independent C++ unit tests.

Known limitations:

- no redistributable OM model is included;
- chunked streaming currently buffers an utterance and decodes only at end;
- the OM graph input/output contract must be checked against the selected export;
- CANN compilation and real OM execution have not run in this environment;
- all board metrics remain unverified until generated by the Atlas workflow.

### 05 — Integration testing: partial

Implemented:

- file and batch inference scaffolding;
- CER, RTF, and latency aggregation;
- a deterministic 1200-case test matrix;
- presentation charts.
- local Web dashboard with result-history API;
- live microphone capture with energy endpointing and a WAV simulation mode.

Not yet verified:

- physical microphone behavior under vehicle noise;
- 1200 corresponding audio files;
- execution results for all planned cases;
- real vehicle-noise VAD accuracy;
- long-duration Dashboard/runner operation.

## Next acceptance gates
