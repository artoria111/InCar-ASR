### 核心接口设计

```cpp
// asr_engine.h — 统一的ASR引擎接口
class ASREngine {
public:
    // 初始化：加载OM模型，配置NPU设备
    bool Init(const std::string& model_path, int device_id = 0);

    // 对PCM16k音频进行识别，返回文本
    std::string Recognize(const std::vector<int16_t>& pcm_data);

    // 流式接口：送入音频片段，返回增量识别结果
    std::string RecognizeStream(const std::vector<int16_t>& pcm_chunk, bool is_end);

    // 性能指标查询
    PerfMetrics GetMetrics();  // RTF, latency, throughput

    // 释放资源
    void Destroy();
};
```

### 各模块技术实现要点

| 模块 | 关键点 | 技术细节 |
|------|--------|----------|
| **VAD Detector** | WebRTC VAD集成 | 设置aggressiveness=2(车载场景)，帧长20ms，逐帧检测，连续3帧非语音才判为静音 |
| **Audio Preprocess** | FBank提取 | 16kHz→25ms窗长/10ms帧移→预加重0.97→汉明窗→512点FFT→80维Mel滤波器→对数压缩 |
| **Ascend Inference** | ACL推理管线 | 动态Batch支持、H2D/D2H异步传输、支持多Stream并发 |
| **CTC Decoder** | 贪心解码 | 移除blank token、合并连续重复token，支持词典约束可选 |

## 四、ONNX → OM 模型转换方案

### ATC转换命令

```bash
# FP16版本（精度优先）
atc --model=paraformer_small.onnx \
    --framework=5 \
    --output=paraformer_small_fp16 \
    --soc_version=Ascend310B1 \
    --input_shape="speech:1,-1,80" \
    --input_format=ND \
    --log=error \
    --enable_small_channel=1 \
    --insert_op_conf=aipp_paraformer.cfg \
    --fusion_switch_file=fusion.cfg

# INT8量化版本（体积优先）
atc --model=paraformer_small_int8.onnx \
    --framework=5 \
    --output=paraformer_small_int8 \
    --soc_version=Ascend310B1 \
    --input_shape="speech:1,-1,80" \
    --input_format=ND \
    --log=error \
    --enable_small_channel=1
```

### 关键参数说明

| 参数 | 说明 |
|------|------|
| `--soc_version=Ascend310B1` | 匹配Atlas 200I DK A2的昇腾310B处理器 |
| `--input_shape="speech:1,-1,80"` | 动态长度维度(-1)，80维FBank特征 |
| `--fusion_switch_file` | 开启Conv+BN+ReLU、MultiHeadAttention+LayerNorm融合 |
| `--enable_small_channel=1` | 针对小模型（<100MB）的性能优化选项 |

## 六、关键技术风险与应对

| 风险 | 概率 | 应对措施 |
|------|:--:|----------|
| ONNX→OM算子不支持 | 中 | ① ONNX Simplifier简化模型图 ② 将不兼容算子从模型中分离为C++预处理 ③ 通过Ascend C开发自定义算子 |
| 动态Shape导致ATC编译失败 | 中 | 先用固定Shape（如`1,512,80`）验证全链路，再逐步扩展为动态维度 |
| MindSpore Lite 2.2.10 与 CANN 7.0.RC1 兼容性 | 低 | 2.2.10是成熟版本，与7.0.RC1兼容；如需使用MindSpore Lite推理备选路径再升级 |
| AscendCL C++开发效率低 | 高 | 先用Python ACL（`/usr/local/Ascend/ascend-toolkit/latest/pyACL/`）快速验证逻辑，再翻译为C++ |
| 推理延迟超标（RTF > 0.2） | 低 | 整图下沉→多Stream并行→减少Beam Size→必要时降级为Paraformer-tiny（~20M） |
| VAD对车载噪声误判 | 中 | 调高aggressiveness到mode=3、增加hang-over延时到500ms、引入能量双门限辅助判定 |
